# Listening Comprehension Trainer

CLI-прототип для тренировки английского через короткий голосовой диалог:
микрофон -> Whisper -> ChatGPT -> TTS.

## Что делает сейчас

- записывает реплику пользователя до момента тишины;
- распознает речь через OpenAI Whisper;
- генерирует ответ преподавателя под уровень `A2`, `B1` или `B2`;
- для запросов про актуальные данные сам запускает интернет-поиск; для погоды использует Open-Meteo, для остальных тем — web search;
- озвучивает ответ через OpenAI TTS с управляемыми паузами между короткими грамматическими chunk-ами;
- как правило заканчивает реплику учителя конкретным вопросом к ученику; если ученик отвечает слишком общо или мимо этого вопроса, учитель должен не просто идти дальше, а помочь выяснить, что именно было не понято, и переформулировать вопрос проще;
- подаёт короткие статусные сигналы: начало и конец записи, тихие напоминания во время ожидания речи, распознавания, генерации ответа и подготовки озвучивания;
- печатает на экран ответ учителя по chunk-ам и показывает паузу между соседними chunk-ами;
- продолжает диалог без подтверждения с клавиатуры между раундами;
- завершает сеанс по `Ctrl+C` или автоматически, если пользователь слишком долго молчит перед новой репликой;
- сохраняет историю диалога в `sessions/`.

## Быстрый старт

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

Для `pyaudio` на Linux/macOS обычно нужен `portaudio`.

### 2. Настроить окружение

```bash
cp .env.example .env
```

Файл `.env` обязателен: `main.py` читает runtime-настройки только из него и больше не держит скрытые fallback-значения в коде.

### 3. Запустить приложение

Интерактивное меню:

```bash
python3 main.py
```

Сразу начать голосовую практику:

```bash
python3 main.py --practice
```

Показать последний сохраненный сеанс:

```bash
python3 main.py --history
```

## Параметры для тюнинга

В `.env` можно менять:

```bash
OPENAI_API_KEY=...
LANGUAGE_LEVEL=B1
PAUSE_THRESHOLD=2.5
PHRASE_TIME_LIMIT=60
SESSION_IDLE_TIMEOUT=45
TTS_SPEED=0.8
TTS_VOLUME=0.9
TTS_MAX_CHUNK_WORDS=5
SMALL_PAUSE_MS=220
CLAUSE_PAUSE_MS=420
SENTENCE_PAUSE_MS=720
```

`SESSION_IDLE_TIMEOUT` задаёт, сколько секунд программа ждёт начала новой реплики.
Если поставить `0`, авто-завершение сеанса по долгому молчанию отключается.
`PAUSE_THRESHOLD` задаёт, сколько тишины внутри ответа считать концом реплики. Если длинный ответ обрезается после короткой заминки, увеличьте это значение.
`PHRASE_TIME_LIMIT` задаёт жёсткий максимум длины одной записываемой реплики. Если поставить `0`, жёсткий лимит отключается. Если ответ обрывается примерно на одном и том же количестве секунд, увеличьте это значение.
`TTS_SPEED` регулирует только скорость произнесения слов.
`TTS_MAX_CHUNK_WORDS` регулирует агрессивность внутреннего chunking: больше значение -> длиннее chunk-и и меньше их количество.
`SMALL_PAUSE_MS`, `CLAUSE_PAUSE_MS` и `SENTENCE_PAUSE_MS` отдельно регулируют паузы между chunk-ами.
Chunking теперь принудительно заканчивается на `, ; : . ! ?` и дополнительно режется на более короткие грамматические части внутри длинных фраз.
Пунктуационные границы обязательны: `TTS_MAX_CHUNK_WORDS` не отключает разрезание на запятых, точках и других знаках.
Эти параметры независимы: можно ускорить слова, не уменьшая паузы, и отдельно ослаблять chunking без смены `LANGUAGE_LEVEL`.
Перед каждым воспроизведением программа явно показывает фактические параметры озвучивания: сервис, скорость слов, число chunk-ов и реальные межchunk-паузы для текущего ответа.
Если любой долгий этап затянулся, программа раз в несколько секунд подаёт короткий статусный сигнал и пишет, что именно сейчас ещё продолжается: запись, распознавание, генерация ответа или подготовка первого TTS chunk-а.
Если пользователь явно просит поискать в интернете или задаёт вопрос про изменяемые данные вроде погоды, новостей, цен, расписаний или часов работы, программа не оставляет решение модели, а сама запускает интернет-поиск по простым правилам.

`.env` теперь является единственным источником этих runtime-настроек. Если обязательный параметр отсутствует или пустой, приложение завершится с явной ошибкой при старте.

Пояснения к ключевым параметрам продублированы прямо в `main.py`.

## Внешние зависимости

- `ffplay` используется для воспроизведения OpenAI TTS WAV-файлов и коротких статусных сигналов;
- если OpenAI TTS или `ffplay` недоступны, ответ остаётся только в текстовом виде.
- для погоды используется публичный API Open-Meteo;
- для общего интернет-поиска используются web results через `ddgs`/DuckDuckGo.

## Ограничения прототипа

- текущий интерфейс только CLI;
- нужен рабочий микрофон;
- нужен OpenAI API key;
- вся логика пока собрана в `main.py`, без разбиения на отдельные сервисы.
